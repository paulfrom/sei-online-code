# Project Memory

> 本文件由平台 seed 初始化生成（origin: platform_seed），是项目记忆的默认起点，不是项目成员主动确认的长期事实。
> 项目成员维护本文件后，应将 front matter 的 reviewStatus 改为 reviewed。
> 优先级：project-memory.md 为最高优先级（P0），高于项目规范、代码现状和平台默认规则。

## Hard Rules

不可违反的项目硬性约束。例如：技术栈限定、目录边界、提交规范、CI 必过项。
生成 PRD / 概览设计 / 详细设计时，任何方案都不得破坏 Hard Rules。

- 待项目维护：列出本项目的 Hard Rules。

## Current Reality

当前代码现状的事实陈述。只描述"现在是什么"，不描述"应该是什么"。
代码事实作为 RealityClaim，不被项目记忆覆盖；规范与现状冲突进入 ConflictFinding。

- 待项目维护：列出当前代码现状关键事实。

## Preferred Direction

项目倾向的技术方向与选型偏好。当存在多种可行方案时，优先选择这里记录的方向。

- 待项目维护：列出本项目偏好方向。

## Forbidden Choices

明确禁止的选型或做法。生成设计时不得出现这里列出的选择。

- 待项目维护：列出本项目禁止的选择。

## Domain Terms

项目领域术语表，统一 PRD / 设计 / 代码中的命名，避免同义异名。

- 待项目维护：列出领域术语及其定义。

## Open Questions

尚未确认、需要人或上级澄清的问题。生成设计时应显式列出并标为待确认，不得静默假定答案。

- 待项目维护：列出待确认问题。
